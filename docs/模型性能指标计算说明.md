# 模型性能指标计算说明

## 概况

评估分四层：窗口级 → 事件级 → 时间级 → 患者级。当前评估报告 v3 涵盖全部四层。

---

## 一、窗口级指标（train.py 训练时）

**粒度**：单个 600 心搏窗口

**前置**：模型对每个窗口输出一个 AFib 概率 (0~1)，与窗口标签对比。

```
二元化阈值: 标签 > 0.1 视为阳性 (y_true_binary = (y_true > 0.1).astype(int))

TP = 标签≥0.8 且 预测≥0.1
TN = 标签<0.8 且 预测<0.1
FP = 标签<0.8 且 预测≥0.1
FN = 标签≥0.8 且 预测<0.1
```

| 指标 | 公式 | 含义 | 代码位置 |
|---|---|---|---|
| AUROC | ROC 曲线下面积，遍历所有阈值 | 整体二分类判别力 | train.py:247 |
| AUPRC | PR 曲线下面积 | 对不平衡数据更敏感 | train.py:248 |
| Brier | `mean((pred - true)²)` | 概率校准度，越低越好 | train.py:249 |
| Train Loss | 加权 BCE + Focal + FP惩罚 | 训练目标 | train.py:292-297 |
| Val Loss | 同 Train Loss (无梯度) | 泛化能力 | train.py:314-319 |

**代码** (train.py:241-250):
```python
def evaluate_clinical_metrics(y_true, y_pred):
    y_true_binary = (y_true > 0.1).astype(int)
    auroc = roc_auc_score(y_true_binary, y_pred)
    auprc = average_precision_score(y_true_binary, y_pred)
    brier = brier_score_loss(y_true_binary, y_pred)
    return auroc, auprc, brier
```

**重要注意**：窗口级 AUROC ≈ 0.98 不等于临床假阳性低。窗口级看的是逐窗口分类，事件级看的是告警管线输出。

## 二、事件级指标（评估报告 [1]）

**粒度**：单个 AFib 发作事件 / 单个 AI 告警事件

### 2.1 前置处理

#### GT 合并
两个 AFib 发作的 120min 预警窗口有重叠 → 合并为一个临床事件：
```python
merged_gt = []
for gt in sorted(gt_episodes, key=lambda g: g['start']):
    if merged_gt and (gt['start'] - 120) <= merged_gt[-1]['end']:
        merged_gt[-1]['end'] = max(merged_gt[-1]['end'], gt['end'])
    else:
        merged_gt.append(gt)
```

#### 告警匹配（一告警一命中）
每个 GT 事件匹配一个最佳告警（选 EWT 最大者），一个告警只能匹配一个 GT：
```python
for gt in merged_gt:
    best_alarm_idx = None
    best_ewt = -999
    for idx, alarm in enumerate(ai_alarms):
        if idx in used_alarm_indices:
            continue
        if overlap(alarm, [gt.start - 120, gt.end]):
            ewt = gt.start - max(alarm.start, gt.start - 120)
            if ewt > best_ewt:
                best_ewt = ewt
                best_alarm_idx = idx
    if best_alarm_idx is not None:
        caught += 1
        used_alarm_indices.add(best_alarm_idx)
```

### 2.2 核心公式

```
TP (True Positive)  = 成功匹配到 GT 的告警数 = 被命中的 GT 数
FP (False Positive) = 未匹配任何 GT 的告警数
FN (False Negative) = 未被任何告警命中的 GT 数

Sensitivity / Recall = TP / (TP + FN) = Caught / Total_GT
Precision / PPV      = TP / (TP + FP) = Caught / Total_Alarms
F1 Score             = 2 × Sens × Prec / (Sens + Prec)
FAR (event)          = FP / Total_Hours × 24
```

### 2.3 提前预警 vs 发作检测

```
EWT (Early Warning Time) = max(0, GT_start - alarm_start), cap 120min

提前预警 (EWT > 0): 告警先于 AFib 发作触发的次数
发作检测 (EWT = 0): 告警在 AFib 发作中才触发的次数

提前预警率 = EWT>0 的命中数 / 总命中数
```

### 2.4 误报统计

```python
for idx, alarm in enumerate(ai_alarms):
    if idx in matched_alarm_indices:  # 已命中 GT 的不算误报
        continue
    is_false_alarm = True
    for gt in merged_gt:
        if overlap(alarm, [gt.start - 120, gt.end]):
            is_false_alarm = False
            break
    if is_false_alarm:
        false_alarms += 1
```

## 三、时间级指标（评估报告 [2]）

**粒度**：分钟

| 指标 | 公式 | 含义 |
|---|---|---|
| AFib 时间占比 | total_afib_mins / total_mins × 100 | 数据集整体的 AFib 负荷 |
| AFib 覆盖率 | afib_covered_mins / total_afib_mins × 100 | 发作时长中被告警覆盖的比例 |
| 告警负担 | total_alarm_mins / total_mins × 100 | 告警占监控总时间的百分比 |
| NSR 告警时间 | (total_alarm_mins - afib_covered_mins) / (total_mins - total_afib_mins) × 100 | 正常心律时告警占比（时间级 FAR） |

**计算代码**：
```python
total_alarm_mins = sum(max(0, a['end'] - a['start']) for a in ai_alarms)
total_afib_mins = sum(gt['end'] - gt['start'] for gt in merged_gt)

# AFib 覆盖
for gt in merged_gt:
    for idx in matched_alarm_indices:
        overlap = min(alarm.end, gt.end) - max(alarm.start, gt.start)
        if overlap > 0:
            afib_covered_mins += overlap

# NSR 告警时间
nsr_mins = total_mins - total_afib_mins
nsr_alarm_mins = total_alarm_mins - afib_covered_mins
nsr_far_time = nsr_alarm_mins / nsr_mins × 100
```

## 四、患者级指标（评估报告 [4]）

**粒度**：单患者

| 指标 | 计算方式 |
|---|---|
| Per-patient Sens | 每个患者独立算 Sens = Hit / GT，再统计所有患者分布的 med/p25/p75/min/max |
| Alarm% | 该患者总告警时长 / 总监控时长 × 100 |
| 状态标记 | MISS (Hit=0), PART (Hit<GT), FA=N (Hit=GT 但 FA>3), OK (Hit=GT 且 FA≤3) |

**代码**：
```python
patient_sens = []
for p in patient_reports:
    gt = p['gt_episodes_count']
    if gt > 0:
        patient_sens.append(p['caught_episodes_count'] / gt * 100)

med = np.median(patient_sens)
p25 = np.percentile(patient_sens, 25)
p75 = np.percentile(patient_sens, 75)
```

## 六、指标的天然缺陷与注意事项

### 6.1 事件级 → 掩盖长发作覆盖不足

1次3小时AFib只要被命中1个窗口就算Sens=100%，即使其余2小时59分都没覆盖。  
→ 这就是为什么 AFib 覆盖率常远低于 Sensitivity（如 30% vs 94%）。

### 6.2 事件级 → 长短 FA 被等权计数

1次持续3小时的误报和1次30秒的误报在事件级都只算1次FA。  
→ NSR告警时间% 才能反映真实假阳性负担。

### 6.3 窗口级 ≠ 事件级

AUROC=0.98 不意味着事件级 Precision 高。窗口级看的是逐窗口二分类，  
事件级取决于告警状态机的时序行为（触发/退出/冷却）。

### 6.4 事件级 Sens 受 merge_gap 影响

GT 合并策略直接影响分母。merge_gap=30心跳时 299碎片，  
merge_gap=120min临床窗口时仅35事件，Sens从片面低值变为真实值。

### 6.5 Per-patient 中位数掩盖离群

Sens median=100% 但仍有 MISS ALL 患者（00735）。  
患者级分布统计算 min/p25/median/p75/max 才能看到全景。
