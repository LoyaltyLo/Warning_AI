# 基于文献调研与测试结果的优化建议

## 当前状态

| 指标 | MIT-BIH | SYAF |
|---|---|---|
| Sensitivity | 94.3% | 84.1% |
| Precision | 26.4% | 10.5% |
| F1 | 41.3% | 18.7% |
| NSR 告警时间 | 77.5% | 32.4% |
| EWT>0 | 55% | 65% |

**核心目标**：保持 Sens≥90%，将 Prec 提升到 50%+，F1 提升到 60%+，FAR 降到 3/24h 以下。

---

## 一、文献发现的关键策略

### 1. 两阶段验证（Two-Stage Verification）⭐⭐⭐

**来源**: Doggart et al. "Two-Stage Classifier for MCOT Alarm Optimization"; Chocron et al. "ArNet2" (*npj Digital Medicine*, 2023)

**方法**: 第一阶段高灵敏度触发 → 第二阶段轻量 CNN/规则验证 → 只有两阶段都通过才正式告警

**效果**: PPV 提升 >11%，灵敏度损失 <5%。128K 参数的二阶段验证器即可。

**对照我们的系统**: 当前 `_adaptive_alert` 触发即告警，无验证环节。可加入"待确认"中间态：
```
触发 → PENDING(等2窗口) → 概率仍高于确认阈值 → ALARM
                         → 概率回落 → 取消(IDLE)
```

### 2. 患者特异性阈值（Patient-Specific Thresholding）⭐⭐⭐

**来源**: ArNet2 — 根据患者 AFib 负荷选择不同的 GRU 后处理单元; Kumar et al. "DeepAware" (2022)

**方法**: 根据前 1 小时告警密度动态调整阈值。高 FA 患者自动抬高阈值。

**对照我们的系统**: 当前 `_compute_adaptive_thresholds` 仅用前 30 窗口校准，shift 范围 ±0.05。SYAF 的高 FA 患者（如 H2409023WVI3F，1 GT vs 58 FA）需要明显更高的阈值，但自适应系统无法做到这一点。

**实现方案**:
```python
# 运行中根据 FA 密度调整阈值
if recent_fa_rate > 10_per_hour:
    shift += 0.10   # 大幅抬高
elif recent_fa_rate > 5_per_hour:
    shift += 0.05   # 适度抬高
```

### 3. 分割式检测 + 滑动窗口多数投票（Segmentation + Voting）⭐⭐

**来源**: Feng et al. (2024, *ACM ICBIP*) — 1D UNet 分割 + 64-RRI 窗口多数投票

**方法**: 不是逐窗口二分类，而是将概率序列视为分割问题。每个窗口被多次评估（如滑动窗口重叠），取多数投票结果。减少孤立假阳性。

**对照我们的系统**: 当前 P1_sustain=4、P2_count≥2 本质上是窗口计数，但可以更系统化：
```
对每个时间点，聚合前后 ±2 窗口的概率 → 取中位数 → 平滑后的概率序列
```

### 4. 信号质量感知门控（Signal Quality-Aware Gating）⭐⭐

**来源**: SQUWA (CHIL 2024); ArNet2 (2023)

**方法**: 对低信号质量窗口降低权重或直接排除。ArNet2 排除 bSQI<0.8 的窗口，丢弃 >75% 低质量窗口的患者。

**对照我们的系统**: 已有 SQA 门控（`extract_features` 中 RR 范围/差分检查），但仅做"通过/拒绝"二元判断。可增加连续质量评分并融入告警决策：
```python
quality_score = 1.0 - (diff_spike_ratio / 0.20)  # 归一化质量评分
adjusted_prob = prob * quality_score              # 低质量窗口概率打折
```

### 5. P 波 / 形态学特征去伪（Morphology-Based False Positive Rejection）⭐⭐

**来源**: DeepAware (Kumar et al., 2022) — P 波检测区分 AFib 和 PVC/PAC 假阳性

**方法**: 即使 RR 间期不规则（类似 AFib），如果 P 波存在且规律，则不是 AFib。QRS 形态变化也能辅助判断。

**对照我们的系统**: `ecg_morphology.py` 的 3 个 QRS 形态特征完全未用。这是最大的未利用资源。

## 二、结合测试结果的针对性问题

### 问题 A：MIT-BIH NSR 告警时间 77.5%（告警粘滞）

根因：持续性 AFib 患者退出阈值太低，告警在 AFib 结束后不退出。

**文献启发**：两阶段验证的"待确认"态可以反向使用——告警退出时也加确认：
```
ALARM → 概率低于 exit 阈值 → PENDING_EXIT(等3窗口) → 确认概率持续低 → IDLE
                                                     → 概率回升 → 保持 ALARM
```

### 问题 B：SYAF 少数患者 FA 极端（4 人贡献 182 次 FA）

根因：统一阈值无法适应这些患者的 NSR 模式（模型对其输出持续 0.5+ 概率）。

**文献启发**：患者特异性阈值 + 信号质量门控。这些患者可能有独特的 NSR 模式或信号质量问题。

### 问题 C：事件级 Prec=26%，但窗口级 AUROC=0.98

根因：告警状态机将零星的窗口级假阳性放大为完整告警事件。

**文献启发**：分割式多数投票 + 两阶段验证。在触发前增加确认环节，过滤孤立假阳性。

## 三、优化建议（按优先级）

### P0 — 两阶段告警确认（预期 Prec +10-15pp，Sens -2-3pp）

```
告警触发 → PENDING(2窗口确认期) → 概率持续高于确认阈值 → 正式 ALARM
                                 → 概率回落 → 丢弃(不算FA)
```

- 实现位置：`_adaptive_alert` 增加 PENDING 中间态
- 确认窗口：2 个（~1 分钟）
- 确认阈值：P1_enter（与入场阈值相同）
- 预期效果：过滤 ~30% 的碎片化 FA，Sens 损失 <3%

### P1 — 运行中 FA 密度自适应（预期 SYAF FA -20-30%）

- 追踪过去 1 小时的 FA 次数
- FA > 10/h：所有阈值 +0.10（大幅抬高）
- FA > 5/h：所有阈值 +0.05（适度抬高）
- FA < 2/h：恢复默认阈值
- 实现位置：`_compute_rolling_thresholds`

### P2 — 启用 ecg_morphology 形态学特征（预期 Sens +3-5pp, Prec +5-8pp）

- QRS 模板相关性 + QRS 离群率 + 导联间相关下降
- Input 16D → 19D，需重新训练

### P3 — 告警退出确认（预期 MIT-BIH NSR 告警 -20pp）

- 退出时也加 3 窗口确认，避免短暂概率波动导致反复触发

### P4 — 分割式多数投票平滑（预期 FA -10-15%）

- 每个窗口取前后 ±2 窗口中位数，平滑概率序列
